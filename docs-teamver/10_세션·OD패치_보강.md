## 3. 세션·인증 보강

### 3.1 401 정책 통일 (P0)

**문제:** `/auth/session`은 만료·invalid 토큰에도 HTTP 200을 반환한다. FE는 `authenticated: false`만 보고 login redirect를 하지 않는다.

**수정 — `deploy/teamver/be/app/routers/auth.py`**

```python
# Before (L56–57): bootstrap 실패 → _empty_session() (200)
# After:
async def get_auth_session(request: Request) -> Any:
    token = extract_request_access_token(request)
    if not token:
        return _empty_session()  # 200 — 쿠키 없음 = 비로그인 (정상)

    try:
        bootstrap = await fetch_bootstrap(token)  # optional → required
    except AuthenticationError:
        raise HTTPException(status_code=401, detail="session_expired")

    return { "authenticated": True, "user": ..., "workspaces": ... }
```

**규칙**

| 조건 | HTTP | body |
|------|------|------|
| 토큰 없음 | 200 | `{ authenticated: false }` |
| 토큰 invalid/expired | **401** | `{ detail: "session_expired" }` |
| 토큰 valid | 200 | `{ authenticated: true, user, workspaces }` |

`/bootstrap`과 동일한 `fetch_bootstrap` + `raise_http_for_teamver_error` 패턴을 사용한다.

### 3.2 refresh 경로 (P0)

**문제:** `@teamver/app-sdk` 기본 refresh URL은 BFF base (`design-api.teamver.com/api/v1/auth/refresh`)인데 design-api에 해당 라우트가 없다.

**권장: SDK → Main BE 직결**

`apps/web/src/teamver/designBffClient.ts`에서 `TeamverClient` 생성 시:

```typescript
import { resolveTeamverMainApiBaseUrl } from "./designApiBase";

const client = new TeamverClient({
  baseUrl: resolveDesignApiBaseUrl(),       // BFF
  refreshUrl: `${resolveTeamverMainApiBaseUrl()}/api/auth/refresh`,  // Main BE
  appKey: "design",
  withCredentials: true,
  tokenStore: null,
  onAuthExpired: () => {
    window.location.href = resolveTeamverLoginUrl();  // same-tab
  },
});
```

**`designApiBase.ts`에 추가할 함수**

```typescript
export function resolveTeamverMainApiBaseUrl(): string {
  const host = window.location.hostname.toLowerCase();
  if (host.includes("stg")) return "https://stg-api.teamver.com";
  if (host.endsWith(".teamver.com")) return "https://api.teamver.com";
  return import.meta.env.VITE_TEAMVER_MAIN_API_URL ?? "http://127.0.0.1:8000";
}
```

**대안 (차선):** design-api `POST /api/v1/auth/refresh` → Main BE `/api/auth/refresh` proxy + Set-Cookie relay. cookie domain 이슈로 **Main BE 직결이 더 단순**.

### 3.3 design-api nginx 게이트 (P0)

**문제:** `design-api.teamver.com`은 nginx `auth_request` 밖에 있어 BFF가 공개 노출된다.

**수정 — `deploy/teamver/devops/nginx/stg-design.teamver.com.https.conf` (design-api server block)**

```nginx
# design-api.teamver.com server { ... }

# 공개 허용 (cookie 자체 검증)
location = /api/v1/auth/session { ... proxy_pass design-api; }
location = /healthz { ... }

# 보호 (로그인 필수)
location /api/v1/usage/ {
    auth_request /_teamver_auth;
    error_page 401 403 = @teamver_login;
    proxy_pass http://teamver_design_api;
}
location /api/v1/projects/ {
    auth_request /_teamver_auth;
    error_page 401 403 = @teamver_login;
    proxy_pass http://teamver_design_api;
}
location /api/v1/bootstrap {
    auth_request /_teamver_auth;
    error_page 401 403 = @teamver_login;
    proxy_pass http://teamver_design_api;
}
```

Production `design.teamver.com.http.conf` design-api 블록에도 동일 적용.

**주의:** CORS preflight `OPTIONS`는 auth_request 제외.

### 3.4 nginx auth_request_set — daemon identity (P1, Phase 3 연동)

**2026-06-16 진행:** Main BE `GET /api/auth/session-check`가 검증된 JWT 기준
`X-Teamver-User-Id`, `X-Teamver-Workspace-Id`를 204 응답 헤더로 반환한다. design nginx는
`auth_request_set`으로 해당 헤더를 받아 OD daemon `/api/`와 design-api 보호 라우트에 전달한다.
남은 작업은 staging 배포 후 실제 요청에서 header propagation을 확인하는 것이다.

**design-api 사용 조건:** `TRUST_TEAMVER_PROXY_HEADERS=true`일 때만 design-api가
`X-Teamver-User-Id`, `X-Teamver-Workspace-Id`를 AuthContext로 신뢰한다. 이 값은 nginx
`auth_request` 뒤 내부 upstream에서만 켜고, public 직접 노출 환경에서는 기본값(`false`)을 유지한다.

**목표:** daemon `/api/projects/:id/*` 호출 시 user/workspace 검증에 사용.

**Option A — Main BE session-check 확장 (권장)**

`ns-teamver-be/src/router/auth.py` session-check 응답에 헤더 추가:

```python
@router.get("/session-check", status_code=204)
async def session_check(user: jwtDataDTO = Depends(...)) -> Response:
    return Response(
        status_code=204,
        headers={
            "X-Teamver-User-Id": user.user_id,
            "X-Teamver-Workspace-Id": user.workspace_id or "",
```

```nginx

    proxy_set_header X-Teamver-Workspace-Id $auth_workspace_id;
    proxy_set_header Authorization "Bearer $teamver_od_api_token";
```

### 3.6 CSRF 방어 (P1)

design-api FastAPI 미들웨어 추가 (`deploy/teamver/be/app/middleware/csrf.py`):

```python
ALLOWED_ORIGINS = {"https://design.teamver.com", "https://stg-design.teamver.com", ...}

async def verify_origin_middleware(request: Request, call_next):
    if request.method in ("POST", "PUT", "PATCH", "DELETE"):
        origin = request.headers.get("origin", "")
        site = request.headers.get("sec-fetch-site", "")
        if site == "cross-site" and origin not in ALLOWED_ORIGINS:
            return JSONResponse(status_code=403, content={"detail": "origin_not_allowed"})
    return await call_next(request)
```

**v1:** Origin + Sec-Fetch-Site 검증. **v2:** CSRF token (double-submit cookie).

SameSite=Lax cookie + CORS `allow_credentials` + origin whitelist로 대부분 CSRF를 차단한다.

## 9. Auth Slide Hardening (2026-07-03)

embed 세션·BYOK·export 실사용 gap 일괄 보강. 상세 changelog — [00 §2026-07-03](./00_구현_내역_누적.md).

### 9.1 세션 만료·복귀 경로 (`returnTo`)

| 경로 | 이전 | 이후 |
|------|------|------|
| `session.authenticated === false` (200) | `redirectToDesignLogin` + `storeAuthReturnTo` | 동일 |
| **401 NetworkError** (`useTeamverEmbed`, BFF `onAuthExpired`) | `redirectToTeamverLogin()` — **returnTo 누락 → `/` 복귀** | `redirectToTeamverLoginPreservingRoute({ returnTo })` |

**코드:** `designAuthFlow.ts` · `useTeamverEmbed.ts` · `designBffClient.ts`  
**테스트:** `teamver-login-preserving-route.test.ts` · `teamver-use-embed.test.tsx`

### 9.2 멀티탭 workspace / session sync

`dispatchTeamverWorkspaceChanged` · `setTeamverEmbedSessionAuthenticated` 가 tab-local CustomEvent만 쏘면 다른 탭이 최대 ~5min drift.

- **`BroadcastChannel('teamver-embed')`** — peer tab에 post → 수신 tab에서 동일 CustomEvent 재발행
- **storage fallback** — `teamver_design_active_workspace_id` · `teamver_design_broadcast_last`
- **ProjectView guard (hotfix):** workspace-changed 시 **ID가 바뀐 경우에만** BYOK stream detach — 동일 ID relay로 onboarding question-form 중 조기 중단 방지
- **App guard (hotfix):** embed boot 때 현재 workspace id를 ref에 먼저 주입해 동일 workspace relay를 switch로 오인하지 않음. workspace switch 후 현재 프로젝트가 recent/list page에 없더라도 **single registry `/projects/{id}` direct check가 성공하면** 홈 이동과 “이 워크스페이스에서 해당 프로젝트에 접근할 수 없습니다” 토스트를 억제.
- **Registry guard (hotfix):** registry list miss는 stale/partial일 수 있으므로 접근 거부 전 `/projects/{id}` direct lookup으로 재확인. 성공 시 FE access cache와 S3 prefix cache를 prime.
- **Session-trusted project guard (hotfix):** 생성 직후 `pendingLocalProjectIds` 또는 현재 세션의 `sessionActiveRunProjectIds`에 포함된 프로젝트는 route mount/open/deep-link/workspace switch guard에서 먼저 허용한다. registry/list 전파 지연 때문에 백그라운드 슬라이드 생성·수정 중인 프로젝트가 홈으로 튕기거나 접근 불가 토스트를 띄우는 것을 방지한다.
- **Background re-entry UI/poll guard (hotfix):** reattach된 running turn은 `live` recovery phase로 표시하고 chat stream UI와 inset banner를 중복 노출하지 않는다. BYOK proxy-stream poll transient 실패는 `/api/runs` polling 전체를 reject시키지 않으며, 해당 poll에서 active background tracking을 삭제하지 않는다.
- **Question-form send guard (hotfix):** 완성된 미답변 question-form turn은 reattach pending과 분리한다. `conversationAwaitingQuestionFormAnswer()`로 유저 입력 대기 중 composer/질문 폼 submit을 막지 않고, BYOK recovery effect도 해당 turn에서 skip한다.
- **Recent list workspace guard (hotfix):** 홈 최근 프로젝트 `inflight` dedupe는 `workspaceId + limit`로 분리하고, App project-list request token은 시작 시점 workspace id를 보관한다. 응답 적용 시 active workspace가 달라졌으면 stale list를 무시해 이전 workspace 목록이 현재 workspace를 덮지 않게 한다.
- **Streaming pseudo-tool guard (hotfix):** streaming chunk 경계에서 `<tools>`, `<invoke>` 등이 나뉘어 들어와도 `createBufferedTextUpdates()` flush 시점에 누적 content/text event를 다시 sanitize한다. 2차 보강으로 화면에 저장된 정제본이 아니라 **원본 스트림 누적본**을 별도 보존해 sanitize하고, `</invoke>`, `</tools>`, `</tool_call_chunk>` 같은 orphan close tag도 contracts SSOT에서 제거한다. persisted load sanitizer 전에 화면에 먼저 노출되는 경로를 막는다.
- **Session relay (hotfix):** cross-tab은 **`authenticated:true` (로그인)만** mirror — peer `false`는 각 탭이 자체 session probe로 확인 (다른 탭 cold boot 실패가 실행 중 탭을 끊지 않음)

**코드:** `teamverEmbedBroadcast.ts` · `teamverWorkspaceEvents.ts` · `teamverEmbedSession.ts`

### 9.3 session_unreachable backoff · orphan JWT

| 항목 | 동작 |
|------|------|
| **C1 backoff** | BFF probe `session_unreachable` → visible tab에서 5s→15s→60s auto `refresh({ force:true })` |
| **C2 orphan JWT** | BFF refresh 400/401 + `user_not_found` / `token.user_not_in_database` → Main BE `/api/auth/logout` (cookie clear, fire-and-forget) |
